Code Spire / Event Modeling

The review pipeline as a timeline

Actors and their screens sit in swimlanes up top. Time runs left to right along the axis. A command (blue) is an intent; it produces an event (red) on the stream; a view (green) folds events for a screen. The automation gears are the sagas that react to an event and fire the next command -- which is almost the whole pipeline here.

Reading one slice (S2, the command pattern). The automation gear reacts to PullRequestEventReceived and fires the RequestReview command; the ReviewLifecycle aggregate turns it into ReviewRequested on the stream; the ReviewStatus view folds that and it appears on the operator dashboard. S1 and S7 are translations -- a Bitbucket webhook becomes an event with no command. Solid lines are "produces"; dotted lines "inform / are displayed on". Diffs are re-fetched by commit, so DiffFetched carries metadata only. The rightmost slice (S9) is cancellation: merging or declining the PR fires PullRequestClosed, the automation issues CancelReview, the aggregate records ReviewCancelled, and in-flight workers abandon at their next stage boundary -- no LLM spend completes, no comment lands on a closed PR (ADR-013).
